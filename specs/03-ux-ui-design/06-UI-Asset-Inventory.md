## Purpose

This document lists the icons that can be implemented with `react-native-vector-icons` and the non-icon visual assets that must be prepared before building each generated screen.

Recommended icon policy:

- Use `MaterialCommunityIcons` as the only icon family for all app UI icons. It is broad enough for navigation, playback, audio, consent, offline, error, and progress states.
- Treat the ShadowSpeak waveform mark, lesson artwork, level illustrations, hero illustrations, ad creative, and app/avatar imagery as real assets, not icon-font replacements.
- iOS/Android status bar symbols and the bottom home indicator are system UI, not app assets.

| Audio waveform visual                       | Lesson detail preview, practice, comparison, recordings, error illustrations         | For static previews, can be generated by app UI; for brand/hero illustrations, prepare vector/PNG artwork.                                                                          |
| Lesson thumbnails                           | Home, catalog, detail, practice, downloads, progress, recording library, audio error | Served via CDN URL from `Lesson.thumbnailUrl`. Client caches by topic key in app data; downloads once per topic on first encounter. See lesson image inventory below for reference. |
| User avatar placeholder                     | Profile, account                                                                     | Can be generated by UI from initials; no image needed unless custom avatar upload is added.                                                                                         |
| App icon / notification icon                | Local reminder notification and OS surfaces                                          | Prepare platform app icon set and notification small icon.                                                                                                                          |

## Export Preparation Notes

- Lesson thumbnails are **CDN assets referenced by `Lesson.thumbnailUrl`**, not static assets to prepare locally. The backend returns the URL keyed by `topic`; the client downloads to app data on first encounter and caches by topic key for reuse across lessons.
- Export reusable lesson covers separately from screen mockups; avoid cropping them from UI screenshots.
- Keep hero error illustrations as individual assets so each error screen can load independently.
- Keep all vector-icon usage in a central `Icon` wrapper/component, but map every icon to `MaterialCommunityIcons` for MVP consistency.
- For waveform UI inside players and recordings, prefer drawing/generating bars in React Native instead of exporting many static waveform images.
- For OAuth buttons, use official Google and Apple brand assets and follow their brand usage guidelines.
